Heat Pump EN 15316-4-2 Implementation Audit

A traceable crosswalk between EN 15316-4-2:2008, the pyBuildingEnergy heat-pump code, and the generated result files. This page is written for reviewing the standard, source code, and outputs side by side.

Audit workflow: keep this page open next to src/pybuildingenergy/source/heat_pump_15316_4_2.py and the user-supplied DIN EN 15316-4-2:2008-09 PDF. Search the code by function name and search the standard by the clause or annex labels given below.

Scope And Boundary

Implemented Standard Path

The implementation follows the detailed temperature-bin method for heat-pump generators in EN 15316-4-2:2008, clause 5.3. It calculates generation energy for space heating and DHW from thermal requirements, source/sink temperatures, product capacity data, COP maps, auxiliary energy, storage losses, and backup limits.

System Boundary

The calculator starts at generator input. It expects the useful heat/cooling demand already calculated by building and distribution methods. The example script obtains space heating/cooling from ISO 52016 and DHW from the package DHW calculation.

Cooling Treatment

EN 15316-4-2:2008 is a heating and DHW generation standard. The code adds reversible space cooling as an explicit extension using the same bin and product-map pattern, with EER and SEER outputs reported separately.

Main Calculation Steps And Formulas

The following formulas are the code formulas. They are written with pyBuildingEnergy variable names so an auditor can connect them directly to heat_pump_bin_results.csv and the functions listed in the crosswalk.

1. Normalize Time Series

Function: _prepare_timeseries()

Q_H_gen_out_kWh, Q_W_gen_out_kWh, Q_C_gen_out_kWh = demand columns converted to kWh per time step
T_H_sink_C = heating_curve(T_ext_C) if no heating sink temperature is supplied

2. Assign Outdoor-Temperature Bins

Functions: _simulate_bins(), _simulate_one_bin(). Standard audit pointer: EN 15316-4-2:2008, clause 5.2 BIN method.

bin_lower_C = floor(T_ext_C / bin_width_C) * bin_width_C
bin_upper_C = bin_lower_C + bin_width_C
hours = sum(time_step_hours)
effective_hours = hours * (24 - utility_cutoff_hours_per_day) / 24

3. Aggregate Service Demand And Losses Per Bin

Function: _service_load_and_performance().

Q_gen_out = sum(Q_service_i)
Q_gen_ls = loss_kWh_per_day * max(T_storage_avg - T_ambient, 0) / storage_test_deltaT_K * hours / 24
Q_total_required = Q_gen_out + Q_gen_ls

For cooling, storage losses are zero in the current implementation.

4. Split Load Above Heat-Pump Operating Limit

Function: _operating_limit_backup_fraction().

DHW: f_backup = (T_W_target_C - hp_operating_limit_C) / (T_W_target_C - T_W_cold_C)
Heating: f_backup = (T_H_sink_C - hp_operating_limit_C) / (T_H_sink_C - T_H_return_C)
Q_backup_operating = Q_total_required * clip(f_backup, 0, 1)

The fraction is zero when the target/sink temperature is within the heat-pump operating limit.

5. Interpolate Product Capacity And COP/EER

Functions: _interpolate_performance(), _bilinear_or_linear().

COP_or_EER = interp2(performance_map, T_source_C, T_sink_C)
capacity_kW = interp2(performance_map, T_source_C, T_sink_C, "capacity_kW")

Rectangular product maps use bilinear interpolation. If the supplied map is scattered, inverse-distance interpolation is used.

6. Allocate Runtime, Heat-Pump Output, Backup, And Unmet Load

Functions: _allocate_heating_and_dhw_runtime(), _allocate_single_heating_service(), _allocate_cooling_runtime().

Q_hp_requested = Q_total_required - Q_backup_operating - Q_unmet_operating
required_runtime_h = Q_hp_requested / capacity_kW
Q_hp_out = min(Q_hp_requested, capacity_kW * available_hours)
Q_backup_capacity = max(Q_hp_requested - Q_hp_out, 0) when backup is available
Q_unmet = capacity_shortfall when backup_mode == "none" or cooling backup is omitted

For alternative heating/DHW operation, DHW is allocated first by default and heating receives the remaining effective runtime.

7. Calculate Compressor, Backup, Environmental, And Rejected Energy

Functions: _finalize_heating_service(), _allocate_cooling_runtime().

Heating/DHW: E_hp_in = Q_hp_out / COP
Heating/DHW: E_backup_in = Q_backup_out / backup_efficiency
Heating/DHW: Q_environment_in = max(Q_hp_out - E_hp_in, 0)
Cooling: E_hp_in = Q_hp_out / EER
Cooling: Q_rejected = Q_hp_out + E_hp_in

8. Calculate Auxiliary Energy And Recoverable Losses

Function: _simulate_one_bin().

W_aux_active = external_auxiliary_power_W * hp_runtime_h / 1000
W_aux_standby = standby_power_W * max(hours - hp_runtime_h, 0) / 1000
W_gen_aux = W_aux_active + W_aux_standby
Q_gen_aux_ls = W_gen_aux * auxiliary_loss_to_ambient_fraction
Q_gen_aux_ls_rbl = Q_gen_aux_ls * (1 - auxiliary_location_b_factor)
Q_storage_ls_rbl = Q_storage_ls * (1 - storage_location_b_factor)
Audit point: heating/DHW auxiliaries and cooling auxiliaries are reported separately. If one physical reversible unit should have only one annual standby budget across all modes, review whether the current separate WHW_gen_aux_kWh and WC_gen_aux_kWh treatment matches the intended accounting boundary.

9. Summarize Seasonal Outputs

Function: _summarize().

EHW_gen_in = EH_hp_in + EW_hp_in + EHW_backup_in
EC_gen_in = EC_hp_in + EC_backup_in
SPF_HW_gen = (QH_gen_out + QW_gen_out) / (EHW_gen_in + WHW_gen_aux)
SPF_HW_hp = (QH_hp_out + QW_hp_out) / (EH_hp_in + EW_hp_in + WHW_gen_aux)
SEER_C_gen = QC_gen_out / (EC_gen_in + WC_gen_aux)
e_HW_gen = 1 / SPF_HW_gen

Audit Checklist

  1. In the standard, read the scope and generator boundary. Confirm that this module starts from generator heat output demand and does not include building, emission or distribution calculations.
  2. Read clause 5.2. Confirm that the typology method is intentionally not implemented.
  3. Read clause 5.3. In code, inspect _simulate_bins() and confirm the bin construction and bin-hour totals.
  4. Compare the standard operating-condition inputs to _prepare_timeseries(): source temperature, sink temperature, return/cold temperature and heat output demand.
  5. Compare product data requirements to _coerce_performance_map() and _interpolate_performance(). Confirm whether your study uses manufacturer/test/default data.
  6. Check _allocate_heating_and_dhw_runtime() against the intended combined operation: independent, alternative, DHW priority, utility cutoffs.
  7. Check _hp_available() and _operating_limit_backup_fraction() against the intended monovalent/bivalent and maximum-temperature behavior.
  8. Check storage loss assumptions in _storage_loss(). Decide whether the simplified daily-loss model is adequate for the project.
  9. Check WHW_gen_aux_kWh and WC_gen_aux_kWh accounting if heating and cooling belong to the same physical unit.
  10. Compare heat_pump_bin_results.csv row by row against the formulas on this page, then compare heat_pump_summary.csv against _summarize().

Known Limits And Deliberate Simplifications

Area Current Treatment Audit Implication
Official default data No official national annex tables are embedded. Use manufacturer data, measured data, or approved national annex defaults for formal studies.
Part-load degradation Part-load behavior is only represented if it is already reflected in the supplied COP/capacity map. Add explicit part-load correction if the audit requires it and the standard/national annex defines the needed factors.
Source-side pumps/fans Handled only through generic auxiliary power inputs. Check whether source-side electricity is included in product data or should be added as external auxiliary power.
Combustion-engine or absorption heat pumps The code currently reports electricity-style input energy and COP/EER maps. Fuel-driven and absorption-specific accounting would need additional input/output terms.
Cooling Implemented as a reversible EER-bin extension, not as a normative EN 15316-4-2:2008 heating/DHW result. Keep cooling outputs separate when making claims about EN 15316-4-2 compliance.
Distribution/emission/storage subsystem standards Not part of this generator module. Use the relevant EN 15316 parts upstream if full system compliance is required.
Primary energy Not calculated here. Feed generator energy outputs into the relevant overall energy/performance calculation.
