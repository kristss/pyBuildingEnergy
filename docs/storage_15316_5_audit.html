EN 15316-5:2017 Heating/DHW Storage Implementation Audit

This page maps the implemented storage module to BS EN 15316-5:2017, using the local file BS EN 15316-5_2017 Energy performance of buildings. Method for calculation of system energy requirements and system.pdf. The implemented scope is Method B, the single-volume storage model, for space-heating and domestic hot water storage systems. Cooling storage is not in the scope of EN 15316-5.

Implemented Boundary

In the example chain, EN 15316-5 is placed between distribution and generation: 

ISO 52016 needs
  -> EN 15316-2 emission
  -> EN 15316-3 distribution
  -> EN 15316-5 heating/DHW storage
  -> EN 15316-4-2 heat pump generation

Therefore Q_H_sto_out_kWh and Q_W_sto_out_kWh are the loads requested downstream of the storage. The module adds storage losses, subtracts auxiliary heat recovered in the medium, and returns Q_H_sto_in_kWh and Q_W_sto_in_kWh as the generator-side input loads.

Main Formulae

Thermal storage losses, from Formula (30):

Q_sto_ls = H_sto_ls * f_sto_bac_acc * f_sto_dis_ls * max(theta_sto_set - theta_sto_amb, 0) * t_ci / 1000

Standby loss coefficient from a declared daily standby loss, following Annex A/B:

H_sto_ls = Q_stby_ls_ref * 1000 / (24 * (theta_sto_set_ref - theta_amb_ref))

Storage charging/output pump runtime, following the logic of Formulae (31) and (32):

t_pmp = Q_transferred / (rho_cp_water * Vdot_pmp * Delta_theta_pmp)

Auxiliary electricity, Formula (33):

W_sto_aux = t_input_pmp * P_input_pmp + t_output_pmp * P_output_pmp

Recoveries, Formulae (34) to (37):

Q_sto_aux_rvd = W_sto_aux * f_rvd_aux
Q_sto_aux_rbl = W_sto_aux * (1 - f_rvd_aux) * f_sto_m
Q_sto_ls_rbl = Q_sto_ls * f_sto_m
