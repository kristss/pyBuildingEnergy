EN 12831-3:2017 DHW Implementation Audit

This page maps the domestic hot-water energy-need calculation in src/pybuildingenergy/source/DHW.py to BS EN 12831-3:2017. It is intended to be read together with the standard, the code and the generated example outputs.

Scope. The implementation now covers both EN 12831-3 DHW energy-need characterization and the DHW system design-heat-load/sizing procedure. The sizing calculator builds the one-minute needs curve, calculates storage switch points, storage/distribution losses, heat-generator time lag, effective reheating power and the supply curve. Distribution, storage and generation losses for the annual simulation are still calculated downstream by EN 15316-3, EN 15316-5 and EN 15316-4-2.

Main Formulas

Residential Equivalent Persons

For residential dwellings the standard derives a maximum equivalent-person count from useful or habitable floor area, then applies the reduced equivalent-person expression for larger dwellings.

n_P_eq = n_P_eq_max if n_P_eq_max < 1.75 else 1.75 + 0.3 * (n_P_eq_max - 1.75)

Code: calc_V_w_day_through_V_w_p_day()

Daily Delivered DHW Volume

For the table method, liters per person per day are taken from the selected residential typology and multiplied by the equivalent-person count. The reference volume is then corrected from reference hot/cold temperatures to the requested draw temperature.

V_W_nd_d = V_ref * (theta_h_ref - theta_c_ref) / (theta_draw - theta_c_ref)

Code: Volume_and_energy_DHW_calculation()

Daily Energy Need

The delivered volume is converted to energy using water density, specific heat capacity and the draw-to-cold-water temperature difference.

Q_W_nd_d = V_W_nd_d * rho_w * c_w * (theta_draw - theta_cold)

Code: Volume_and_energy_DHW_calculation()

Hourly Allocation

Hourly fractions for workdays, weekends and holidays are normalized over the annual calendar so the sum of hourly energy equals the annual DHW energy need.

f_x_avg = total_days / (n_workdays * sum_x_workday + n_weekends * sum_x_weekend + n_holidays * sum_x_holiday)

Code: x_q_h_i_coor, Q_W_nd_h_i and hourly_cons_energy

Storage Capacity and Switch Points

Q_sto,max = V_sto * rho_w * c_w * (theta_sto,max - theta_cold) * f_l
Q_sto,on = Q_sto,max * (1 - h_sensor / h_sto)
Q_sto,min = V_sto * rho_w * c_w * (theta_draw - theta_cold) * f_l * (1 - h_sensor / (2 * h_sto))

Q_sto,min is used for mixed storage systems. For loading storage systems the minimum margin is zero, following Clause 6.4.2.4.2.

Losses, Lag and Effective Reheat

The DHW design calculator first looks for explicit project or product data via standby_loss_kWh_per_day or q_sb_sto_kWh_d. If neither is provided, q_sb,sto is interpolated from Annex B Table B.8 for the active storage volume. For example, 180 l gives 1.35 + (180 - 150) / (200 - 150) * (1.56 - 1.35) = 1.476 kWh/day.

Q_W,sto,t = q_sb,sto * (theta_sto,max - theta_a) / 45 / 1440
Q_W,dis,t = sum(U_dis,j * l_dis,j * (theta_m,j - theta_a,j)) / 60000
t_lag = t_lag,HG + t_lag,dis
Phi_eff = Phi_N - Phi_W,sto - Phi_W,dis

For mixed storage systems the effective heat-exchanger power is reduced as the mean storage temperature approaches the charging temperature, using the exponential storage-temperature expression in Formula (15).

Supply-Curve Criterion

margin_t = Q_sto,residual,t - Q_sto,min

The design is reported as satisfied when the minimum margin over the one-minute representative design day is non-negative within the configured tolerance. With --dhw-sizing-mode size_storage, the example increases DHW storage volume until this criterion is met.

Optional External References

The implementation uses EN 12831-3 Annex B defaults where no product or national data are provided. For project-specific direct-flow pipe sizing and fixture design-flow methods, EN 806-3 can be used as an input source. For manufacturer storage standby-loss declarations, EN 12897 or product regulations can provide q_sb,sto. For single-family tapping programs, EN 13203-2 can be used instead of the Annex B/default hourly profile.
